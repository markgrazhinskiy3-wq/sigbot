Нужно доработать только аналитический движок, потому что после усиления фильтров бот почти перестал выдавать сигналы.

Проблема:
Сейчас логика слишком строгая и слишком часто возвращает no-signal.
Нужно ослабить правила так, чтобы бот снова начал выдавать сигналы, но без мусорных входов.

Главная цель:
- не ломать архитектуру
- не переписывать бота целиком
- изменить только signal engine и связанные модули анализа
- сделать баланс между качеством и частотой сигналов

Что нужно изменить:

1. Снизить жесткость signal scoring engine
Сейчас движок слишком строгий.
Нужно сделать 3 уровня результата:
- strong signal
- moderate signal
- no signal

Рекомендуемая логика:
- confidence >= 68 -> strong signal
- confidence >= 58 and < 68 -> moderate signal
- confidence < 58 -> no signal

2. Не требовать совпадения всех стратегий одновременно
Сигнал можно выдавать, если:
- сработала хотя бы одна основная стратегия:
  - impulse_pullback_strategy
  - level_bounce_strategy
  - false_breakout_strategy
и одновременно:
- candle_strength не слабый
- market_regime не chaotic_noise
- нет жесткого запрета по уровню

Не нужно требовать одновременного подтверждения всеми стратегиями.

3. Разделить конфликты на мягкие и жесткие
Сейчас любой конфликт слишком вероятно блокирует сигнал.
Нужно сделать:

Жесткий конфликт:
- основная стратегия buy, но market_regime уверенно bearish
- основная стратегия sell, но market_regime уверенно bullish
- вход прямо в сильный уровень против сделки
- false breakout против выбранного направления

Мягкий конфликт:
- RSI neutral
- stochastic neutral
- momentum слабый
- candle strength неидеальный

Правило:
- жесткий конфликт -> no signal
- мягкий конфликт -> просто понижает confidence, но не запрещает сигнал

4. Ослабить запреты в candle_strength_analysis
Сейчас candle filter вероятно слишком часто рубит сигнал.
Изменить:
- если одна слабая свеча или одна длинная тень против входа -> не запрещать, а снижать score
- запрещать сигнал только если:
  - последние 3 свечи реально хаотичны
  - свечи маленькие и разнонаправленные
  - длинные тени в обе стороны
  - нет внятного импульса

5. Ослабить level_analysis
Изменить логику:
- если цена экстремально близко к сильному уровню против входа -> reject
- если цена просто относительно близко -> penalty to confidence
- если есть запас хода -> boost confidence

Не запрещать слишком много сигналов только из-за близости уровня.

6. Ослабить market_regime_analysis
Сейчас chaotic_noise вероятно определяется слишком часто.
Изменить классификацию так, чтобы:
- chaotic_noise ставился только в реально грязом рынке
- range использовался чаще, чем chaotic_noise
- weak trend использовался отдельно
- trend up / trend down определялись только при реальном подтверждении

Если режим = range, это не автоматический запрет.
В range можно разрешить сигналы типа level_bounce и false_breakout.
В range не запрещать сигнал автоматически.

7. Сделать основную и вторичную логику
Основные модули:
- impulse_pullback_strategy
- level_bounce_strategy
- false_breakout_strategy

Вторичные фильтры:
- candle_strength_analysis
- level_analysis
- market_regime_analysis
- ema/rsi/stochastic/momentum

Правило:
- основная стратегия формирует направление
- вторичные фильтры усиливают или ослабляют confidence
- вторичные фильтры не должны без причины блокировать все сигналы

8. Изменить веса score
Использовать более мягкие веса, например:
- price action strategies: 55%
- candle strength: 15%
- levels: 10%
- market regime: 10%
- indicator confirmation: 10%

Важно:
Если одна основная стратегия уверенно сработала, а фильтры не противоречат критично, сигнал должен выдаваться.

9. Добавить debug breakdown
Нужно вывести в debug mode:
- какая основная стратегия сработала
- какой модуль занизил confidence
- какой именно reject_reason сработал
- какие модули дали neutral
- какие модули дали hard reject

10. Добавить fail-safe режим
Если:
- есть 1 сильная основная стратегия
- нет hard reject
- confidence находится в серой зоне 55-58
то можно выдать moderate signal вместо полного отказа

11. Обновить итоговый формат
Возвращать:
{
  "direction": "buy" | "sell" | "none",
  "signal_quality": "strong" | "moderate" | "none",
  "confidence": 0-100,
  "primary_strategy": "impulse_pullback" | "level_bounce" | "false_breakout" | null,
  "reasons": [...],
  "soft_conflicts": [...],
  "hard_conflicts": [...],
  "reject_reason": null | "..."
}

12. Важно
Не переписывай весь проект.
Покажи только обновленные файлы аналитического движка.
Сделай так, чтобы бот снова начал выдавать сигналы, но не скатывался в random.
Главный принцип:
- prefer quality over noise
- but avoid overblocking
- if one primary strategy is valid and there is no hard conflict, allow the signal
