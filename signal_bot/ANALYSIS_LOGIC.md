# Логика анализа сигналов — текущая версия

## Входные данные

48–53 свечи по 15 секунд каждая (≈ 12–13 минут рынка).  
WS и браузер используют одинаковый `period=15` — кэш мёржится корректно.

---

## Шаг 1 — Семь независимых модулей

Каждый модуль возвращает `buy_score` и `sell_score` от 0 до 100.

---

### 1. Market Regime (режим рынка)

Определяет состояние рынка, чтобы правильно взвешивать паттерны.

**Что считает:**
- EMA(9) и EMA(21): наклон EMA9 за последние 3–8 баров, спред между EMA9 и EMA21
- Структура свингов: делит последние 30 баров на 3 части → Higher Highs/Lows или Lower Highs/Lows
- Бычий % последних 20 свечей
- ATR(14) текущий vs исторический (14–35 баров назад) → `atr_ratio`
- Хаос-индикатор: частота смен направления в последних 10 барах + доля тела к диапазону

**Классификация:**

| Режим | Условие |
|---|---|
| `chaotic_noise` | ATR×1.3+ И EMA нейтральны (<0.03%) И смен направления ≥70% + мелкие тела |
| `uptrend` | ≥60 pts вверх, опережает медведей на 15+ pts |
| `downtrend` | ≥60 pts вниз, опережает быков на 15+ pts |
| `weak_trend` | ≥35 pts в одну сторону, опережает на 8+ pts |
| `range` | все остальные случаи |

**Volatility state:** `compressed` если ATR < 50% исторического, `high` если > 150%, иначе `normal`

---

### 2. Impulse + Pullback (импульс и откат) — v4

Ищет классический паттерн: **импульс → откат → подтверждающая свеча**.

**Сканирование:** последние 25 баров.

**Алгоритм для каждой точки старта:**
1. **Импульс** — 2–6 последовательных баров в одном направлении, средний размер тела ≥ 60% от avg_body
2. **Откат** — 1–3 бара в противоположном направлении, слабее импульса (средний откат < 95% от среднего импульса)
3. **Подтверждение** — следующая свеча **должна закрыться в направлении импульса** (doji/нейтральные запрещены)

**Оценка (0–100):**
```
impulse_q  = avg_impulse_body / (avg_body × 1.2), capped at 1.0
pullback_q = 1 - (avg_pullback / avg_impulse)
recency    = 1 - (bars_since_confirmation / scan_window)
conf_bonus = +0.15 если подтверждающая свеча крупная (> 0.3×avg_body)

raw = (impulse_q×0.40 + pullback_q×0.25 + recency×0.25 + conf_bonus) × 100
```

**Доминантное направление** (фоллбек, если нет чёткого паттерна):
- Требует 70%+ баров в направлении за последние 25
- Из последних 6 баров — минимум 4 в направлении
- Последний бар в направлении
- Результат **ограничен максимумом 24** — не может создать standalone сигнал
  (PARTIAL_MATCH_MIN=30), только усиливает слабый паттерн

---

### 3. Level Bounce (отбой от уровня)

Ищет касание зоны поддержки/сопротивления с отбоем.

**Зона:** уровень ± tolerance, где `tolerance = max(0.2%, 1× avg_body_pct) × price`

**Логика для поддержки (BUY):**
1. Было ли касание зоны в последних 3 барах?
2. Текущая цена в зоне или выше? (+35 pts)
3. Последний бар бычий? (+30) / нейтральный (+10)
4. Нижняя тень > 0.3×avg_body? (+20) / есть тень (+8)
5. Бонус за свежесть: 1 бар назад (+10), 2 бара (+5)
6. Штраф, если сопротивление слишком близко сверху (< 0.1% — ×0.4, < 0.3% — ×0.7)

Аналогично для сопротивления (SELL).

---

### 4. False Breakout (ложный пробой)

Ищет ловушку: цена пробила уровень, но вернулась обратно.

**Для ложного пробоя вниз → BUY:**
1. В последних 5 барах хоть один бар прощупал зону снизу (закрылся ниже — «сильный», тень ниже — «слабый»)
2. Текущая цена вернулась выше зоны
3. Оценка: 35 (база) + бычье закрытие (+30) + крупное тело (+20) + свежесть (+5/бар) + сильный пробой (+10)
4. Пробой > 3 баров назад → ×0.8

Аналогично для пробоя вверх → SELL.

> Контртрендовый штраф на false_breakout **не применяется** — паттерн по природе контртрендовый.

---

### 5. Level Analysis (анализ уровней)

Находит ключевые уровни и оценивает пространство для движения.

**Нахождение уровней:** свинговые хаи/лои с окном 3 бара, кластеризация с допуском 0.3%.

**Оценка headroom (расстояние до противоположного уровня):**

| Расстояние | Скор |
|---|---|
| < 0.02% | 5 — цена вплотную |
| < 0.05% | 18 — экстремально близко |
| < 0.15% | 38 — тесно |
| < 0.40% | 62 — умеренно |
| ≥ 0.40% | 88 — хорошее пространство |

`buy_score` = расстояние до сопротивления выше  
`sell_score` = расстояние до поддержки ниже

В итоговой формуле используется **opposing level score**: для BUY берётся sell_score (насколько далеко сопротивление), для SELL — buy_score.

---

### 6. Candle Strength (сила свечей)

Анализирует последние 5 свечей.

**Что считает:**
- Количество бычьих / медвежьих / дожи
- Средний body_ratio (тело / диапазон) для каждого типа
- `upper_dom` = верхняя тень / avg_body — давление продавцов
- `lower_dom` = нижняя тень / avg_body — давление покупателей

**BUY-скор (база 50):**
- ≥3 бычьих: +30, 2 бычьих: +15, иначе: −10
- bull_strength × 0.2
- lower_dom > 1.5: +12 (покупатели держат снизу)
- upper_dom > 1.5: −15 (продавцы давят сверху)

**Neutral flag** (штраф ×0.55 на оба скора) — только при действительно хаотичных свечах:
- Почти все дожи (≥4/5), ИЛИ
- 2+ дожи + смешанные бычьи/медвежьи с разрывом ≤1, ИЛИ
- Все тела < 12% диапазона + смешанные

---

### 7. Indicator Confirmation (подтверждение индикаторами)

**Роль вторичная** — только подтверждает, не порождает сигнал.

| Индикатор | Логика |
|---|---|
| EMA(9/21) | EMA9 > EMA21 → buy; EMA9 < EMA21 → sell; спред < 0.01% → neutral |
| RSI(14) | < 38 → buy; > 62 → sell; иначе neutral |
| Stochastic(5,3) | < 25 → buy; > 75 → sell; иначе neutral |
| Momentum(10) | цена vs 10 баров назад; изм. > 0.02% → buy/sell |

`buy_score = buy_votes × 25`, `sell_score = sell_votes × 25` (max 100 при 4/4 голосах)

## Шаг 2 — Режимная корректировка паттернов

Перед блендингом к сырым скорам трёх паттернов применяются **бусты по режиму:**

| Режим | Impulse | Bounce | Breakout |
|---|---|---|---|
| uptrend / downtrend | +15 | −10 (если < 70) | −10 (если < 70) |
| range | −10 (если < 50) | +15 | +15 |
| weak_trend / chaotic | без изменений | без изменений | без изменений |

**Блендинг:**
```
pa_score = best_adjusted × 0.70 + avg_of_three_raw × 0.30
```

**Пороги соответствия (по сырому скору лучшего паттерна):**
- `best_raw ≥ 50` → `full_match`
- `best_raw ≥ 30` → `partial_match` (минимум для участия в сигнале)

---

## Шаг 3 — Жёсткие блоки (только 2)

1. Оба направления не имеют паттерна (`buy_partial = False` и `sell_partial = False`) → NO_SIGNAL
2. Паритет — оба активны, но разрыв `|pa_buy_raw − pa_sell_raw| < 3` → NO_SIGNAL

---

## Шаг 4 — Выбор направления

```
uptrend   → BUY   (исключение: pa_sell − pa_buy ≥ 15 → SELL-override)
downtrend → SELL  (исключение: pa_buy − pa_sell ≥ 15 → BUY-override)
range / weak_trend → сторона с бо́льшим pa_score
```

---

## Шаг 5 — Итоговая уверенность (confidence)

```
confidence = pa_score        × 0.58
           + candle_strength × 0.15
           + indicators      × 0.12
           + regime_score    × 0.10
           + opposing_level  × 0.05
```

**Мягкие штрафы (суммируются):**

| Условие | Штраф |
|---|---|
| chaotic_noise режим | −15 |
| weak_trend режим | −8 |
| partial match (не full) | −8 |
| контртренд (не breakout) | −10 |
| opposing level < 38 | −5 |
| neutral_flag свечей | −5 |
| ATR compressed | −4 |

---

## Шаг 6 — Финальное решение

| confidence | Решение |
|---|---|
| ≥ 68 | **STRONG** |
| ≥ 45 | **NORMAL** |
| < 45 | **NO_SIGNAL** |

**Шкала 1–5:**
- ≥ 88 → 5/5
- ≥ 78 → 4/5
- ≥ 68 → 3/5
- ≥ 45 → 2/5

**Рекомендованная экспирация:**
- Bounce / Breakout → 1 мин
- Impulse в чётком тренде → 2 мин
- Остальные → 1 мин
